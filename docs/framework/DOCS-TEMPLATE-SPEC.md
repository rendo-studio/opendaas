# OpenDaaS 共享文档固定页面模板规范
## OpenDaaS Docs Template Specification v1

**版本：** v1.1  
**状态：** 当前主稿  
**产品工作名：** `OpenDaaS`  
**CLI 名：** `opendaas`  
**性质：** 实施规范  
**依赖：** [共享文档目录协议规范](./DOCS-DIRECTORY-SPEC.md)、[共享项目文档包规范](./PROJECT-PACK-SPEC.md)

---

## 1. 文档目的

本文档定义 `docs/` 固定页面的最小模板字段。

它要回答的是：

- 固定锚点页面最少应包含哪些段落
- 这些段落以什么顺序出现最合理
- 哪些字段必填
- 哪些字段可选

---

## 2. 基本判断

固定页面模板的目标不是制造僵硬表单，而是：

1. 保证最小信息完整
2. 让人类阅读路径稳定
3. 让 Agent 快速提取高层上下文
4. 让 `opendaas init/adopt` 能生成有用占位，而不是空白文件

因此，本规范只定义：

- **最小必需区块**
- **建议顺序**
- **可选扩展区块**

不要求所有项目写成完全一样的文风。

---

## 3. 通用模板规则

### 3.1 页面标题必须清晰

每个固定页面必须有明确标题，不使用内部流程对象名称充当人类主标题。

### 3.2 先摘要，再细节

页面应优先提供一眼能看懂的摘要，再进入细节。

### 3.3 可缺省但不可失焦

允许某些字段暂时留空，但不允许页面完全失去其锚点职责。

### 3.4 可读优先于形式完美

段落表达可以自然，但必须回答页面应回答的问题。

### 3.5 头部元信息必须存在

所有固定页面模板都必须包含标准 front matter，且至少包含 `name` 与 `description`。

## 4. `docs/index.md` 模板（可选）

### 4.1 页面职责

这是整个共享文档包的可选入口页。

### 4.2 最小必需区块

建议顺序如下：

1. `项目一句话定义`
2. `最终目标`
3. `当前进度`
4. `项目边界 / 非目标`
5. `快速开始阅读路径`
6. `主要内容导航`

### 4.3 建议模板骨架

```md
---
name: 项目名称
description: 项目入口页，概览最终目标、当前进度与阅读路径。
---

# 项目名称

## 一句话定义

## 最终目标

## 当前进度

## 边界与非目标

## 从哪里开始

## 文档导航
```

### 4.4 可选扩展区块

- 目标用户
- 核心价值
- 当前阶段摘要

## 10. 条件性固定页面模板方向

当以下页面存在时，建议最少包含：

### 10.1 `docs/engineering/deployment.md`

最少区块：

1. 部署目标
2. 环境差异
3. 发布流程
4. 回滚或验证方式

### 10.2 `docs/project/decisions/index.md`

最少区块：

1. 决策摘要
2. 关键决策列表
3. 最近新增决策

### 10.3 `docs/internal/versions/index.md`

最少区块：

1. 版本摘要
2. 最近版本
3. 重要变化

---

## 11. CLI 生成要求

`opendaas init` 或 `opendaas adopt` 至少应：

1. 生成 `docs/shared/overview.md` 与 `docs/shared/goal.md`
2. 为默认生成的锚点页提供最小标题与区块占位
3. 避免只生成一个空标题文件

CLI 生成的目标不是替用户写完整内容，而是确保最小 docs package 具备可继续填写的合理骨架。

---

## 12. 不合格状态

以下情况应视为固定页面模板尚未正确落地：

1. 页面存在，但只有标题没有职责区块
2. 页面区块存在，但完全无法回答该页面应回答的问题
3. 最终目标页缺失或失焦
4. 页面结构充满内部流程字段，普通人无法阅读
5. `init/adopt` 生成的页面只有空白内容，无法继续使用

---

## 13. 当前结论

当前可以明确：

> **固定页面需要稳定模板，但模板只应固定最小区块，不应强制固定全部表达方式。**

更具体地说：

> **入口页、目标页、状态页、当前工作页、changes 页、development 页，都已经可以收敛到最小区块级模板。**
